Setup and initialize new SPI instance.

Setup and create SPI instance according to parameters.

Note
This must be called exactly once per new SPI instance.
Slave mode is not implemented!!!
Returns
handle to the SPI instance created. Should be used as parameter to the send() functions.
Parameters
masterSlaveModeSPI in Master or Slave mode see spi_masterSlaveMode_t
clockDividerdefines the SPI clock divider see spi_clockDivider_t
spiModedefines the SPI mode to use [0..3]
See also
spi_mode_t
spi_mode CPOL CPHA Clock Behavior
0 0 0 Low when idle, samples on leading edge
1 0 1 Low when idle, samples on trailing edge
2 1 0 High when idle, samples on leading edge
3 1 1 High when idle, samples on trailing edge
Parameters
dataOrderSend LSB or MSB first
See also
spi_dataOrder_t
Parameters
*csPortport register for the CS pin - Ex. &PORTB. If no CS is used then NULL
csPinthat is connected to the slaves CS - Ex. PB6. If no CS is used then don't care
csActiveLevel
  • 0: CS active when low.
  • 1: CS active when high.
rxFifoSize0: No receive buffer used, means no data will be received from slave or the size.
txFifoSize0: No transmit buffer used, means that only one byte can be transmitted at the time or the size.
*callBackpointer to a call back function that will be called when the driver receives a byte from the SPI bus. The function should have the following signature:
void handlerName(spi_p spiInstance, uint8_t rxByte)
  • 0: no call back function will be called.
